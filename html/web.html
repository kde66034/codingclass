웹 표준(Web Standards)이란?

브라우저의 종류 및 버전에 따른 기능 차이에 대해 호환이 가능하도록 제시된 표준이다.
다른 기종 또는 플랫폼에 따라 달리 구현되는 기술을 동일하게 구현함과 동시에 어느 한쪽에만 최적화가 치우치지 않도록 공통 요소를 사용해 웹 페이지를 제작하는 기법으로 표준화 단체인 W3C(World Wide Consortium)가 권고한 표준안에 따라 웹 사이트를 작성할 때 이용하는 HTML, CSS, JavaScript 등에 대한 규정을 담고 있으며 웹 사이트의 사용자가 어떤 브라우저나 OS를 사용하더라도 동일한 결과를 보이게 하는 것이 웹 표준의 궁극적 목적이다.

웹 표준의 장점

01. 소스의 통일화로 인해 수정 및 운영 관리가 용이
02. 다양한 브라우저, 단말기, 장애인 지원용 프로그램에서도 대응 가능 → 접근성 향상
03. 논리적이고 효율적으로 작성되어 코드의 양과 용량이 줄어 서버 부담 감소
04. CSS와 HTML의 분리로 유지보수에 들어가는 시간 절약 가능
05. 불필요한 마크업이 최소화되어 페이지 로딩 속도 향상
06. 구 버전 웹 브라우저에서도 콘텐츠가 원활이 표시되고 호환성·운용성 확보 가능
07. 스크린 리더기 등 보조공학 기기 사용자들이 더 정확한 정보를 얻을 수 있음
08. 검색 봇을 통한 효율적 노출 등 검색 엔진 최적화 가능


웹 접근성(Web Accessibility)이란?

제아무리 잘 설계된 웹 사이트라 할지라도 장애를 가진 사람들의 입장에서는 사용하기 불편할 수 있다. 장애를 가진 사용자들도 쉽게 웹에 접근할 수 있도록 설계 및 개발된 것을 웹 접근성이라고 합니다.

웹 접근성의 중요성

현재 국내 웹은 과도한 플러그인 사용 등 웹 표준이 지켜지지 않는 곳이 다수이며 이는 장애인과 노약자를 포함한 모든 사용자들의 웹 브라우저를 통한 정보 접근에 제약을 준다. 따라서 어떤 브라우저를 사용하건 모든 사용자들이 동일한 웹 사이트를 보고 이용할 수 있도록 웹 표준기술 작업이 반드시 필요하다.


웹 호환성(Cross Browsing)이란?

표준 웹 기술을 사용하여 어느 한쪽만의 운영체제나 브라우저에 최적화되거나 종속되지 않도록 공통 요소를 사용하여 웹 페이지를 제작하는 기법이며 웹 사이트 사용 시 어떠한 환경에서도 동일한 결과가 나오도록 의미하는 웹 상호운용성의 개념에 웹 표준의 준수를 포함하는 개념이다.

웹 호환성의 중요성

웹 호환성을 준수함으로 인해 장애인과 고령자 등 정보 소외 계층이 비장애인들과 동등히 정보에 자유롭게 접근할 권리를 누릴 수 있으며 이로 인해 잠재적 계층을 사용자 계층으로 끌어낼 수 있습니다. 실제로 웹 호환성을 준수한 외국의 쇼핑몰이 다양한 고객층을 확보해 매출을 증가시킨 사례가 있으며, 국내의 쇼핑몰에서도 노년층 매출이 꾸준히 증가하는 추세임을 알 수 있습니다.
